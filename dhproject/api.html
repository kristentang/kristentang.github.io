    Why this lesson was difficult:

    The girls seemed to really like the HTML, CSS, and JS lessons, and were able to create so amazing websites. The one lesson that was difficult for the girls was the API lesson. I think APIs are incredibly useful and I’m glad they are included in the curriculum, but I think the lesson was too difficult for our girls to follow. Code alongs have been very helpful for our girls to learn new information, however the code provided by the curriculum did not have very clear explanation of the new additions to the code.

    The lesson also incorporated HTML and JS, which are new concepts to the girls. This added an additional layer to the lesson because the girls had to not only apply a brand new concept, but also do so in a language they are still familiarizing themselves with. This is where clear explanations would have really helped the girls. Because there were so much content, it also would have been nice to differentiate between the concepts/code that the girls had already learned, were related to APIs, and JS code that was new.

    We ended the lesson early because we could tell (from observing them during the lessons and reading their journals) the students were lost and frustrated. Most of the girls ended up copying and pasting from the provided github code, and expressed in their journals how they felt like they did not get much out of doing so, even if they were able to get their code to work.

    (8) APIs for the Future

    Describe the activity



    When we ended the lesson, we wanted to give the girls time to decompress, so we didn’t want to force them to keep thinking about APIs. Instead, we gave them time to continue working on the websites they had been making for the past two days. We knew this concept of the lesson was important, and we knew that when we recreated the lesson we would want to include a similar activity.

    (9) Journals and Wrap Up

    At the end of the day, we took first took time to apologize to the girls for not being as prepared as we should have been. We let them know that we believed the lesson was too complex for a beginner lesson, and that had we been more familiar with the lesson, we should have recognized this earlier on.

    We knew what kind of feedback we would be receiving from the girls in their journals. This was not the first time that we had gotten negative feedback, and from similar experiences, such as during the first Python lesson, we were not scared of the negative feedback because we knew we could rework the lesson.

    The Teaching Team's Lesson

    While we never ended up doing the API project, there are so many possibilities that I knew our girls would have had fun with it if they had been more comfortable with APIs. While there were other projects or parts of lessons that we never had time to finish and understanding APIs was not critical to move on with the rest of the curriculum, we felt it was important to reteach this lesson because we felt like it is such a useful topic to know. We didn’t want the girls to leave feeling defeated by a topic and scared of incorporating it into their final project or intimidated of learning it in the future. I believe this paid off, seeing as three out of five teams incorporated APIs into their final project.

    (describe different aspects of this lesson - main aspect was starting with a much smaller goal, breaking down explanation step by step, code along - had to accept a much smaller end product, also we made the lesson so we were comfortable with the code and weren’t learning it along with them - had to admit that we were not as prepared as we should have been)
